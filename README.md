Powered by Jobtern.

## Your task

Your task brief is in [TASK.md](./TASK.md). Read it fully before writing any code.

## How to submit

1. Fork this repository
2. Put all your work inside the `solution/` directory
3. Complete the task described in your brief
4. Open a pull request **back to this repository** (not your fork)
5. Fill in the PR template honestly — it factors into your evaluation
6. Add the `ready-for-review` label to your PR when your submission is ready
7. Wait for the automated review — you will see feedback posted directly on the PR

## Resubmissions

You have a maximum of **3 submissions** in total. If changes are requested, the `changes-requested` label will be added to your PR automatically.

To request a re-review after pushing your fixes:

1. Push your changes
2. Remove the `changes-requested` label
3. Add the `ready-for-review` label again

Each resubmission carries a score penalty. Use your trials wisely.

## Rules

- Work alone
- Any language, framework, or library is allowed unless your brief specifies otherwise
- Your `README` inside `solution/` must include: how to run it, assumptions you made, and one thing you'd do differently with more time
- Commit history is part of the evaluation — single-commit submissions will be flagged

## Questions

If something in the brief is unclear, open an issue on this repo before starting.
That's the expected channel — it signals how you handle ambiguity on a real team.
