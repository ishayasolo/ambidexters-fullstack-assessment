# Assessment Repository

This repo is your workspace for the Jobtern screening assessment.

## How to submit

1. Fork this repository
2. Complete the task described in your brief
3. Open a pull request **back to this repository** (not your fork)
4. Fill in the PR template honestly — it factors into your evaluation
5. Wait for the automated review — you'll see feedback posted directly on the PR

## Rules

- Work alone during the individual sprint
- Any language, framework, or library is allowed unless your brief specifies otherwise
- Your `README` inside the submission must include: how to run it, assumptions you made, and one thing you'd do differently with more time
- Commit history is part of the evaluation. Single-commit submissions will be flagged.

## Questions

If something in the brief is unclear, open an issue on this repo before starting.
That's the expected channel — it signals how you handle ambiguity on a real team.
